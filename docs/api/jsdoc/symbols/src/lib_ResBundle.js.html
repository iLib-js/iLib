 31 /**
 32  * @class
 33  * Create a new resource bundle instance. The resource bundle loads strings
 34  * appropriate for a particular locale and provides them via the getString
 35  * method.<p>
 36  *
 37  * The options object may contain any (or none) of the following properties:
 38  *
 39  * <ul>
 40  * <li><i>locale</i> - The locale of the strings to load. If not specified, the default
 41  * locale is the the default for the web page or app in which the bundle is
 42  * being loaded.
 43  *
 44  * <li><i>name</i> - Base name of the resource bundle to load. If not specified the default
 45  * base name is "resources".
 46  *
 47  * <li><i>type</i> - Name the type of strings this bundle contains. Valid values are
 48  * "xml", "html", "text", "c", or "raw". The default is "text". If the type is "xml" or "html",
 49  * then XML/HTML entities and tags are not pseudo-translated. During a real translation,
 50  * HTML character entities are translated to their corresponding characters in a source
 51  * string before looking that string up in the translations. Also, the characters "<", ">",
 52  * and "&" are converted to entities again in the output, but characters are left as they
 53  * are. If the type is "xml", "html", or "text" types, then the replacement parameter names
 54  * are not pseudo-translated as well so that the output can be used for formatting with
 55  * the IString class. If the type is "c" then all C language style printf replacement
 56  * parameters (eg. "%s" and "%d") are skipped automatically. If the type is raw, all characters
 57  * are pseudo-translated, including replacement parameters as well as XML/HTML tags and entities.
 58  *
 59  * <li><i>lengthen</i> - when pseudo-translating the string, tell whether or not to
 60  * automatically lengthen the string to simulate "long" languages such as German
 61  * or French. This is a boolean value. Default is false.
 62  *
 63  * <li><i>missing</i> - what to do when a resource is missing. The choices are:
 64  * <ul>
 65  *   <li><i>source</i> - return the source string unchanged
 66  *   <li><i>pseudo</i> - return the pseudo-translated source string, translated to the
 67  *   script of the locale if the mapping is available, or just the default Latin
 68  *   pseudo-translation if not
 69  *   <li><i>empty</i> - return the empty string
 70  * </ul>
 71  * The default behaviour is the same as before, which is to return the source string
 72  * unchanged.
 73  *
 74  * <li><i>basePath</i> - look in the given path for the resource bundle files. This can be
 75  * an absolute path or a relative path that is relative to the application's root.
 76  * Default if this is not specified is to look in the standard path (ie. in the root
 77  * of the app).
 78  *
 79  * <li><i>onLoad</i> - a callback function to call when the resources are fully
 80  * loaded. When the onLoad option is given, this class will attempt to
 81  * load any missing locale data using the ilib loader callback.
 82  * When the constructor is done (even if the data is already preassembled), the
 83  * onLoad function is called with the current instance as a parameter, so this
 84  * callback can be used with preassembled or dynamic loading or a mix of the two.
 85  *
 86  * <li>sync - tell whether to load any missing locale data synchronously or
 87  * asynchronously. If this option is given as "false", then the "onLoad"
 88  * callback must be given, as the instance returned from this constructor will
 89  * not be usable for a while.
 90  *
 91  * <li><i>loadParams</i> - an object containing parameters to pass to the
 92  * loader callback function when locale data is missing. The parameters are not
 93  * interpretted or modified in any way. They are simply passed along. The object
 94  * may contain any property/value pairs as long as the calling code is in
 95  * agreement with the loader callback function as to what those parameters mean.
 96  * </ul>
 97  *
 98  * The locale option may be given as a locale spec string or as an
 99  * Locale object. If the locale option is not specified, then strings for
100  * the default locale will be loaded.<p>
101  *
102  * The name option can be used to put groups of strings together in a
103  * single bundle. The strings will then appear together in a JS object in
104  * a JS file that can be included before the ilib.<p>
105  *
106  * A resource bundle with a particular name is actually a set of bundles
107  * that are each specific to a language, a language plus a region, etc.
108  * All bundles with the same base name should
109  * contain the same set of source strings, but with different translations for
110  * the given locale. The user of the bundle does not need to be aware of
111  * the locale of the bundle, as long as it contains values for the strings
112  * it needs.<p>
113  *
114  * Strings in bundles for a particular locale are inherited from parent bundles
115  * that are more generic. In general, the hierarchy is as follows (from
116  * least locale-specific to most locale-specific):
117  *
118  * <ol>
119  * <li> language
120  * <li> region
121  * <li> language_script
122  * <li> language_region
123  * <li> region_variant
124  * <li> language_script_region
125  * <li> language_region_variant
126  * <li> language_script_region_variant
127  * </ol>
128  *
129  * That is, if the translation for a string does not exist in the current
130  * locale, the more-generic parent locale is searched for the string. In the
131  * worst case scenario, the string is not found in the base locale's strings.
132  * In this case, the missing option guides this class on what to do. If
133  * the missing option is "source", then the original source is returned as
134  * the translation. If it is "empty", the empty string is returned. If it
135  * is "pseudo", then the pseudo-translated string that is appropriate for
136  * the default script of the locale is returned.<p>
137  *
138  * This allows developers to create code with new or changed strings in it and check in that
139  * code without waiting for the translations to be done first. The translated
140  * version of the app or web site will still function properly, but will show
141  * a spurious untranslated string here and there until the translations are
142  * done and also checked in.<p>
143  *
144  * The base is whatever language your developers use to code in. For
145  * a German web site, strings in the source code may be written in German
146  * for example. Often this base is English, as many web sites are coded in
147  * English, but that is not required.<p>
148  *
149  * The strings can be extracted with the ilib localization tool (which will be
150  * shipped at some future time.) Once the strings
151  * have been translated, the set of translated files can be generated with the
152  * same tool. The output from the tool can be used as input to the ResBundle
153  * object. It is up to the web page or app to make sure the JS file that defines
154  * the bundle is included before creating the ResBundle instance.<p>
155  *
156  * A special locale "zxx-XX" is used as the pseudo-translation locale because
157  * zxx means "no linguistic information" in the ISO 639 standard, and the region
158  * code XX is defined to be user-defined in the ISO 3166 standard.
159  * Pseudo-translation is a locale where the translations are generated on
160  * the fly based on the contents of the source string. Characters in the source
161  * string are replaced with other characters and returned.
162  *
163  * Example. If the source string is:
164  *
165  * <pre>
166  * "This is a string"
167  * </pre>
168  *
169  * then the pseudo-translated version might look something like this:
170  *
171  * <pre>
172  * "Ţħïş ïş á şţřïñĝ"
173  * </pre>
174  * <p>
175  *
176  * Pseudo-translation can be used to test that your app or web site is translatable
177  * before an actual translation has happened. These bugs can then be fixed
178  * before the translation starts, avoiding an explosion of bugs later when
179  * each language's tester registers the same bug complaining that the same
180  * string is not translated. When pseudo-localizing with
181  * the Latin script, this allows the strings to be readable in the UI in the
182  * source language (if somewhat funky-looking),
183  * so that a tester can easily verify that the string is properly externalized
184  * and loaded from a resource bundle without the need to be able to read a
185  * foreign language.<p>
186  *
187  * If one of a list of script tags is given in the pseudo-locale specifier, then the
188  * pseudo-localization can map characters to very rough transliterations of
189  * characters in the given script. For example, zxx-Hebr-XX maps strings to
190  * Hebrew characters, which can be used to test your UI in a right-to-left
191  * language to catch bidi bugs before a translation is done. Currently, the
192  * list of target scripts includes Hebrew (Hebr), Chinese Simplified Han (Hans),
193  * and Cyrillic (Cyrl) with more to be added later. If no script is explicitly
194  * specified in the locale spec, or if the script is not supported,
195  * then the default mapping maps Latin base characters to accented versions of
196  * those Latin characters as in the example above.
197  *
198  * When the "lengthen" property is set to true in the options, the
199  * pseudotranslation code will add digits to the end of the string to simulate
200  * the lengthening that occurs when translating to other languages. The above
201  * example will come out like this:
202  *
203  * <pre>
204  * "Ţħïş ïş á şţřïñĝ76543210"
205  * </pre>
206  *
207  * The string is lengthened according to the length of the source string. If
208  * the source string is less than 20 characters long, the string is lengthened
209  * by 50%. If the source string is 20-40
210  * characters long, the string is lengthened by 33%. If te string is greater
211  * than 40 characters long, the string is lengthened by 20%.<p>
212  *
213  * The pseudotranslation always ends a string with the digit "0". If you do
214  * not see the digit "0" in the UI for your app, you know that truncation
215  * has occurred, and the number you see at the end of the string tells you
216  * how many characters were truncated.<p>
217  *
218  *
219  * @constructor
220  * @param {?Object} options Options controlling how the bundle is created
221  */

233         if (options.locale) {
234             this.locale = (typeof(options.locale) === 'string') ?
235                     new Locale(options.locale) :

237         }
238         if (options.name) {
239             this.baseName = options.name;
240         }
241         if (options.type) {
242             this.type = options.type;
243         }
244         this.lengthen = options.lengthen || false;
245         this.path = options.basePath;
246 
247         if (typeof(options.sync) !== 'undefined') {
248             this.sync = !!options.sync;
249         }
250 
251         if (typeof(options.loadParams) !== 'undefined') {
252             this.loadParams = options.loadParams;
253             if (!this.path) {
254                 if (typeof (options.loadParams.root) !== 'undefined') {
255                     this.path = options.loadParams.root;
256                 } else if (typeof (options.loadParams.base) !== 'undefined') {
257                     this.path = options.loadParams.base;
258                 }
259             }
260         }
261         if (typeof(options.missing) !== 'undefined') {
262             if (options.missing === "pseudo" || options.missing === "empty") {
263                 this.missing = options.missing;
264             }
265         }
266     } else {
267         options = {sync: true};
268     }

271 
272     lookupLocale = this.locale.isPseudo() ? new Locale("en-US") : this.locale;

275         locale: lookupLocale,
276         name: this.baseName + ".json",
277         sync: this.sync,
278         loadParams: this.loadParams,
279         root: this.path,
280         callback: ilib.bind(this, function (map) {
281             if (!map) {
282                 map = ilib.data[this.baseName] || {};
283             }
284             this.map = map;
285             if (this.locale.isPseudo()) {
286                 this._loadPseudo(this.locale, options.onLoad);
287             } else if (this.missing === "pseudo") {
288                 new LocaleInfo(this.locale, {
289                     sync: this.sync,
290                     loadParams: this.loadParams,
291                     onLoad: ilib.bind(this, function (li) {
292                         var pseudoLocale = new Locale("zxx", "XX", undefined, li.getDefaultScript());
293                         this._loadPseudo(pseudoLocale, options.onLoad);
294                     })
295                 });
296             } else {
297                 if (typeof(options.onLoad) === 'function') {
298                     options.onLoad(this);
299                 }
300             }
301         })
302     });

305     //if (!this.locale.isPseudo() && JSUtils.isEmpty(this.map)) {
306     //    console.log("Resources for bundle " + this.baseName + " locale " + this.locale.toString() + " are not available.");
307     //}

373     /**
374      * @private
375      * Pseudo-translate a string
376      */
377     _pseudo: function (str) {
378         if (!str) {
379             return undefined;
380         }
381         var ret = "", i;
382         for (i = 0; i < str.length; i++) {
383             if (this.type !== "raw") {
384                 if (this.type === "html" || this.type === "xml") {
385                     if (str.charAt(i) === '<') {
386                         ret += str.charAt(i++);
387                         while (i < str.length && str.charAt(i) !== '>') {
388                             ret += str.charAt(i++);
389                         }
390                     } else if (str.charAt(i) === '&') {
391                         ret += str.charAt(i++);
392                         while (i < str.length && str.charAt(i) !== ';' && str.charAt(i) !== ' ') {
393                             ret += str.charAt(i++);
394                         }
395                     } else if (str.charAt(i) === '\\' && str.charAt(i+1) === "u") {
396                         ret += str.substring(i, i+6);
397                         i += 6;
398                     }
399                 } else if (this.type === "c") {
400                     if (str.charAt(i) === "%") {
401                         var m = this.percentRE.exec(str.substring(i));
402                         if (m && m.length) {
403                             // console.log("Match found: " + JSON.stringify(m[0].replace("%", "%%")));
404                             ret += m[0];
405                             i += m[0].length;
406                         }
407                     }
408 
409                 }
410                 if (i < str.length) {
411                     if (str.charAt(i) === '{') {
412                         ret += str.charAt(i++);
413                         while (i < str.length && str.charAt(i) !== '}') {
414                             ret += str.charAt(i++);
415                         }
416                         if (i < str.length) {
417                             ret += str.charAt(i);
418                         }
419                     } else {
420                         ret += this.pseudomap[str.charAt(i)] || str.charAt(i);
421                     }
422                 }
423             } else {
424                 ret += this.pseudomap[str.charAt(i)] || str.charAt(i);
425             }
426         }
427         if (this.lengthen) {
428             var add;
429             if (ret.length <= 20) {
430                 add = Math.round(ret.length / 2);
431             } else if (ret.length > 20 && ret.length <= 40) {
432                 add = Math.round(ret.length / 3);
433             } else {
434                 add = Math.round(ret.length / 5);
435             }
436             for (i = add-1; i >= 0; i--) {
437                 ret += (i % 10);
438             }
439         }
440         if (this.locale.getScript() === "Hans" || this.locale.getScript() === "Hant" ||
441                 this.locale.getScript() === "Hani" ||
442                 this.locale.getScript() === "Hrkt" || this.locale.getScript() === "Jpan" ||
443                 this.locale.getScript() === "Hira" || this.locale.getScript() === "Kana" ) {
444             // simulate Asian languages by getting rid of all the spaces
445             ret = ret.replace(/ /g, "");
446         }
447         return ret;
448     },

472     /**
473      * @private
474      * Create a key name out of a source string. All this does so far is
475      * compress sequences of white space into a single space on the assumption
476      * that this doesn't really change the meaning of the string, and therefore
477      * all such strings that compress to the same thing should share the same
478      * translation.
479      * @param {null|string=} source the source string to make a key out of
480      */
481     _makeKey: function (source) {
482         if (!source) return undefined;
483         var key = source.replace(/\s+/gm, ' ');
484         return (this.type === "xml" || this.type === "html") ? this._unescapeXml(key) : key;
485     },

487     /**
488      * @private
489      */
490     _getStringSingle: function(source, key, escapeMode) {
491         if (!source && !key) return new IString("");
492 
493         var trans;
494         if (this.locale.isPseudo()) {
495             var str = source ? source : this.map[key];
496             trans = this._pseudo(str || key);
497         } else {
498             var keyName = key || this._makeKey(source);
499             if (typeof(this.map[keyName]) !== 'undefined') {
500                 trans = this.map[keyName];
501             } else if (this.missing === "pseudo") {
502                 trans = this._pseudo(source || key);
503             } else if (this.missing === "empty") {
504                 trans = "";
505             } else {
506                 trans = source;
507             }
508         }
509 
510         if (escapeMode && escapeMode !== "none") {
511             if (escapeMode == "default") {
512                 escapeMode = this.type;
513             }
514             if (escapeMode === "xml" || escapeMode === "html") {
515                 trans = this._escapeXml(trans);
516             } else if (escapeMode == "js" || escapeMode === "attribute") {
517                 trans = trans.replace(/'/g, "\\\'").replace(/"/g, "\\\"");
518             }
519         }
520         if (trans === undefined) {
521             return undefined;
522         } else {
523             var ret = new IString(trans);
524             ret.setLocale(this.locale.getSpec(), true, this.loadParams); // no callback
525             return ret;
526         }

529     /**
530      * Return a localized string, array, or object. This method can localize individual
531      * strings or arrays of strings.<p>
532      *
533      * If the source parameter is a string, the translation of that string is looked
534      * up and returned. If the source parameter is an array of strings, then the translation
535      * of each of the elements of that array is looked up, and an array of translated strings
536      * is returned. <p>
537      *
538      * If any string is not found in the loaded set of
539      * resources, the original source string is returned. If the key is not given,
540      * then the source string itself is used as the key. In the case where the
541      * source string is used as the key, the whitespace is compressed down to 1 space
542      * each, and the whitespace at the beginning and end of the string is trimmed.<p>
543      *
544      * The escape mode specifies what type of output you are escaping the returned
545      * string for. Modes are similar to the types:
546      *
547      * <ul>
548      * <li>"html" -- prevents HTML injection by escaping the characters < > and &
549      * <li>"xml" -- currently same as "html" mode
550      * <li>"js" -- prevents breaking Javascript syntax by backslash escaping all quote and
551      * double-quote characters
552      * <li>"attribute" -- meant for HTML attribute values. Currently this is the same as
553      * "js" escape mode.
554      * <li>"default" -- use the type parameter from the constructor as the escape mode as well
555      * <li>"none" or undefined -- no escaping at all.
556      * </ul>
557      *
558      * The type parameter of the constructor specifies what type of strings this bundle
559      * is operating upon. This allows pseudo-translation and automatic key generation
560      * to happen properly by telling this class how to parse the string. The escape mode
561      * for this method is different in that it specifies how this string will be used in
562      * the calling code and therefore how to escape it properly.<p>
563      *
564      * For example, a section of Javascript code may be constructing an HTML snippet in a
565      * string to add to the web page. In this case, the type parameter in the constructor should
566      * be "html" so that the source string can be parsed properly, but the escape mode should
567      * be "js" so that the output string can be used in Javascript without causing syntax
568      * errors.
569      *
570      * @param {?string|Array.<string>=} source the source string or strings to translate
571      * @param {?string|Array.<string>=} key optional name of the key, if any
572      * @param {?string=} escapeMode escape mode, if any
573      * @return {IString|Array.<IString>|undefined} the translation of the given source/key or undefined
574      * if the translation is not found and the source is undefined
575      */
576     getString: function (source, key, escapeMode) {
577         if (!source && !key) return new IString("");
578 
579         //if (typeof(source) === "object") {
580             // TODO localize objects
581         //} else
582 
583         if (ilib.isArray(source)) {
584             return source.map(ilib.bind(this, function(str) {
585                return typeof(str) === "string" ? this._getStringSingle(str, key, escapeMode) : str;
586             }));
587         } else {
588             return this._getStringSingle(source, key, escapeMode);
589         }
590     },

592     /**

594      * the getString() method, but it returns a regular Javascript string instead of

596      * method without being wrapped in an IString instance first.
597      *
598      * @param {?string|Array.<string>=} source the source string to translate
599      * @param {?string|Array.<string>=} key optional name of the key, if any
600      * @param {?string=} escapeMode escape mode, if any
601      * @return {string|Array.<string>|undefined} the translation of the given source/key or undefined
602      * if the translation is not found and the source is undefined
603      */
604     getStringJS: function(source, key, escapeMode) {
605         if (typeof(source) === 'undefined' && typeof(key) === 'undefined') {
606             return undefined;

608         //if (typeof(source) === "object") {

610         //} else
611 
612         if (ilib.isArray(source)) {
613             return this.getString(source, key, escapeMode).map(function(str) {
614                return (str && str instanceof IString) ? str.toString() : str;
615             });
616         } else {
617             var s = this.getString(source, key, escapeMode);
618             return s ? s.toString() : undefined;
619         }
620     },
621 
622     /**
623      * Return true if the current bundle contains a translation for the given key and
624      * source. The
625      * getString method will always return a string for any given key and source
626      * combination, so it cannot be used to tell if a translation exists. Either one
627      * or both of the source and key must be specified. If both are not specified,
628      * this method will return false.
629      *
630      * @param {?string=} source source string to look up
631      * @param {?string=} key key to look up
632      * @return {boolean} true if this bundle contains a translation for the key, and
633      * false otherwise
634      */
635     containsKey: function(source, key) {
636         if (typeof(source) === 'undefined' && typeof(key) === 'undefined') {
637             return false;
638         }
639 
640         var keyName = key || this._makeKey(source);
641         return typeof(this.map[keyName]) !== 'undefined';
642     },
643 
644     /**
645      * Return the merged resources as an entire object. When loading resources for a
646      * locale that are not just a set of translated strings, but instead an entire
647      * structured javascript object, you can gain access to that object via this call. This method
648      * will ensure that all the of the parts of the object are correct for the locale.<p>
649      *
650      * For pre-assembled data, it starts by loading <i>ilib.data[name]</i>, where
651      * <i>name</i> is the base name for this set of resources. Then, it successively
652      * merges objects in the base data using progressively more locale-specific data.
653      * It loads it in this order from <i>ilib.data</i>:
654      *
655      * <ol>
656      * <li> language
657      * <li> region
658      * <li> language_script
659      * <li> language_region
660      * <li> region_variant
661      * <li> language_script_region
662      * <li> language_region_variant
663      * <li> language_script_region_variant
664      * </ol>
665      *
666      * For dynamically loaded data, the code attempts to load the same sequence as
667      * above, but with slash path separators instead of underscores.<p>
668      *
669      * Loading the resources this way allows the program to share resources between all
670      * locales that share a common language, region, or script. As a
671      * general rule-of-thumb, resources should be as generic as possible in order to
672      * cover as many locales as possible.
673      *
674      * @return {Object} returns the object that is the basis for this resources instance
675      */
676     getResObj: function () {
677         return this.map;
678     }
